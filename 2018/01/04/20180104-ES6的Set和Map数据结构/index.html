ES6的Set和Map数据结构

发表于 2018-01-04

2018年开始第一篇博客，这半个多月都没什么事情，自己琢磨了下零售云和地产云的项目后，想自己搞搞清楚整个的搭建过程，学习之余试试把信息发布系统搞起来，但是为什么没写这些还是选择继续看看ES6呢？答案就是：因为我还不会！对，就是理直气壮，好了，现在开始。。。ES6提供的新的数据结构：Set 和 Map

Set

Set类似于数组，但是成员值都是唯一的，没有重复的值(所以可以用来去除数组重复成员的方法)，它本身是一个构造函数，可以用来生成数据实例，或者接受一个数组为参数来初始化一个实例：

注：使用Set添加的值不会发生类型转换，所以5和“5”是两个不同的值（但是所有NaN都是相等的，两个对象{}都是不相等的）

1、Set实例属性和方法

  • Set.prototype.size：返回Set实例的成员总数
  • add(value)：添加某个值，返回Set结构本身
  • delete(value)：删除某个值，返回一个布尔值，表示删除是否成功
  • has(value)：返回一个布尔值，表示该值是否为Set的成员
  • clear()：清除所有成员，没有返回值。
    注： Array.from方法可以将Set结构转为数组,这就为数组去重提供了另一种方式

2、遍历操作
Set结构的遍历方法有四个：

  • keys()：返回键名的遍历器
  • values()：返回键值的遍历器（由于Set结构没有键名，所以该方法和keys方法行为一致，且默认遍历执行的是values）
  • entries()：返回键值对的遍历器
  • forEach()：使用回调函数遍历每个成员
    注：Set的遍历顺序是按照插入顺序的，这个特性有时非常有用，比如使用Set保存一个回调函数列表，调用时就能保证按照添加的顺序调用

WeakSet

WeakSet 结构与Set类似，也是不重复的值的集合。但是，它与Set有两个区别：

  • 首先，WeakSet的成员只能是对象，而不能是其他类型的值；
  • 其次，WeakSet中的对象都是弱引用，即垃圾回收机制不考虑WeakSet对该对象的引用，也就是说，如果其他对象都不再引用该对象，那么垃圾回收机制会自动回收该对象所占用的内存，不考虑该对象还存在于WeakSet之中，所以它适合临时存放对象，以及存放跟对象绑定的信息，只要这些对象在外部消失，它在WeakSet里面的引用就会自动消失（因此，WeakSet成员不适合引用）。

1、语法

  • WeakSet是一个构造函数，一样可以使用new命令创建一个数据结构，同时接受一个数组或类似数组的对象作为参数，该数组的所有成员（注：不是该数组本身）都会自动成为该实例对象的成员，由于WeakSet的成员只能是对象，所有传进来的数组参数的成员必须是对象（数组也行）;
  • WeakSet.prototype.add(value)：向 WeakSet 实例添加一个新成员;
  • WeakSet.prototype.delete(value)：清除 WeakSet 实例的指定成员;
  • WeakSet.prototype.has(value)：返回一个布尔值，表示某个值是否在 WeakSet 实例之中
  • WeakSet 没有size属性，所以没有办法遍历它的成员(因为成员都是弱引用，随时可能消失，遍历机制无法保证成员的存在，很可能刚刚遍历结束，成员就取不到了。WeakSet 的一个用处，是储存 DOM 节点，而不用担心这些节点从文档移除时，会引发内存泄漏。)

Map

已知js对象的本质是键值对的集合，但是传统的只能用字符串当作键，这给它的使用带来了很大的限制，为了解决这个问题，ES6提供了Map数据结构，它也是键值对的集合，但是“键”的范围更广，各种类型都可以，是一种更完善的Hash结构。
作为构造函数，Map也可以接受一个数组作为参数，该数组成员是一个个表示键值对的数组：

Map结构同样有以下几个属性和方法：

  • size属性，返回Map结构的成员总数
  • set(key,value)(set方法返回的是当前的Map对象，因此可以采用链式调用)
  • get(key)
  • has(key)
  • delete(key)
  • clear()清除所有成员，没有返回值
  • keys()，返回键名的遍历器
  • values()返回键值的遍历器
  • entries()返回所有成员的遍历器
  • forEach()遍历Map的所有成员（还可以接受第二个参数，用来绑定this）

  • Map结构转为数组结构比较快速的方法是使用扩展运算符...

  • 结合数组的map方法和filter方法，可以实现Map的遍历和过滤

WeakMap

WeakMap结构和Map结构类似，也是用于生成键值对的集合，但是与Map也有区别：

  • 首先，WeakMap只接受对象作为键名
  • 其次，它的键名所指向的对象，不计入垃圾回收机制（也就是说WeapMap里面的键名对象和所应用的键值对会自动消失，不用手动删除引用）
  • WeakMap与Map在API上的区别主要是，一是没有遍历操作（没有key()、values()、entries()、size属性），只有get()、set()、has()、delete()方法。
